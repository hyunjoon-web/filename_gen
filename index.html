    사용 가이드

    이 도구는 무엇인가요?

    파일명을 미리 정의한 변수명 리스트에 따라 일괄 변환하고 ZIP으로 다운로드하는 도구입니다. 서버 없이 브라우저 메모리에서만 동작해 파일이 외부로 전송되지 않습니다.

    파일명 구조
    prefix + 변수명 + .확장자
    ────────────────────
    badge-streak_1.svg
    badge-lesson_1.svg
    badge-unit_3.svg
    두 가지 모드
    A
    1 → N 복사
    원본 파일 1개를 변수명 개수만큼 복사해 각각 다른 이름으로 저장. 동일한 플레이스홀더 이미지가 필요할 때 적합.
    B
    N → N 매칭
    여러 파일을 업로드 순서대로 변수명 순서에 1:1 매칭하여 각각 리네임. 파일마다 내용이 다를 때 사용.
    언제 쓰나요?

    배지 플레이스홀더처럼 동일한 이미지 파일 하나를 여러 이름으로 복사해야 할 때 사용합니다.

    사용 방법
    1
    상단 탭에서 1 → N 복사 선택
    2
    원본 파일 1개 업로드
    드래그&드롭 또는 클릭해서 선택. 1개만 업로드됩니다.
    3
    Prefix · 변수명 · 확장자 입력
    변수명은 한 줄에 하나씩 입력. 확장자는 원본과 달라도 됩니다.
    4
    미리보기 확인 후 ZIP 다운로드
    변수 개수만큼 파일이 생성됩니다.
    결과 예시
    원본: placeholder.png  ·  prefix: badge-  ·  ext: svg
    ────────────────────────────
    placeholder.png → badge-streak_1.svg
    placeholder.png → badge-streak_2.svg
    placeholder.png → badge-lesson_1.svg
    ... 변수 개수만큼 반복
    언제 쓰나요?

    각기 다른 파일들을 정해진 이름 규칙으로 순서대로 리네임할 때 사용합니다. 업로드 순서 = 변수명 순서.

    사용 방법
    1
    상단 탭에서 N → N 매칭 선택
    2
    파일 여러 개를 순서대로 업로드
    업로드 순서가 곧 매칭 순서입니다. 드래그로 한 번에 올리면 OS 파일 정렬 순서를 따릅니다.
    3
    변수명을 같은 순서로 입력
    파일 1번 ↔ 변수 1번, 파일 2번 ↔ 변수 2번 순으로 매칭됩니다.
    4
    개수 일치 확인 후 ZIP 다운로드
    개수가 다르면 다운로드 시 경고창이 표시됩니다.
    결과 예시
    prefix: badge-  ·  ext: svg
    ────────────────────────────
    1.svg → badge-streak_1.svg
    2.svg → badge-streak_2.svg
    3.svg → badge-lesson_1.svg
    ⚠
    파일과 변수 개수가 다르면 다운로드 시 경고창이 나타납니다. 그래도 진행을 누르면 매칭된 쌍만 ZIP에 포함됩니다.
    주요 기능
    📋
    클립보드 복사
    미리보기 헤더의 복사 버튼을 누르면 변환된 파일명 전체를 클립보드에 복사합니다. Google 스프레드시트에 붙여넣기(⌘V)하면 각 셀에 세로로 입력됩니다.
    🗂
    Prefix / 확장자 설정
    파일명 앞에 붙는 Prefix와 확장자를 자유롭게 설정할 수 있습니다. 확장자를 비워두면 원본 확장자가 유지됩니다.
    🔒
    완전 로컬 처리
    업로드한 파일은 브라우저 메모리에만 임시 저장됩니다. 서버 전송이 없으며 페이지를 닫으면 즉시 사라집니다.
    유용한 팁
    💡
    변수명 리스트를 Google 스프레드시트에서 관리한다면, 열 전체를 복사해 textarea에 붙여넣으면 됩니다. 빈 줄은 자동으로 무시됩니다.
    💡
    Prefix를 비워두면 변수명.확장자 형태로만 저장됩니다. 예: streak_1.svg
    💡
    N→N 모드에서 파일을 드래그&드롭할 때, 한 번에 여러 파일을 선택하면 파일 탐색기의 정렬 순서를 따릅니다. 순서가 중요하다면 미리보기를 꼭 확인하세요.
    💡
    GitHub Pages에서 이 도구를 사용 중이라면, 도구 자체는 GitHub 서버에서 HTML만 내려받는 것이며 파일은 전혀 전송되지 않습니다.
